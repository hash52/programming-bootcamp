## 1. AIにSQLを書かせてみよう

最近はChatGPTなどのAIに「こういうデータが欲しい」と伝えれば、SQLを書いてもらえる時代になった。

例えば、こんな依頼をしてみよう。

> **あなた:** 「社員テーブルから、部門ごとの平均給与を出して」
> **AI:** はい。以下のSQLで取得できます。

```sql
SELECT dept_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY dept_id;
```

一見、便利に見える。しかし、ここで問題がある。

### 概念を知らないとAIに指示できない

もし、あなたが **「GROUP BY」という概念を知らなかったら** どうなるだろうか。

- 「部門ごと」という要件を、そもそもAIに伝えられない
- AIが生成したSQLの `GROUP BY` が正しいかどうか判断できない
- 結果が期待通りかどうかも検証できない

つまり、 **SQLの構文を暗記する必要はないが、概念の理解は必須** ということである。

## 2. AIに任せられない3つのこと

AI時代でも、データベースについて人間が理解しておくべきことがある。

### 1. 要件を正確に伝える

AIにSQLを書かせるには、 **何が欲しいかを正確に伝える** 必要がある。
そのためには、データベースの概念（テーブル、結合、集計など）を理解していなければならない。

| やりたいこと | 必要な概念理解 |
|-------------|--------------|
| 「部門ごとの社員数を出して」 | GROUP BY（グループ化） |
| 「社員と部門名を一緒に表示して」 | JOIN（テーブル結合） |
| 「給与が30万以上の社員だけ抽出して」 | WHERE（条件指定） |

### 2. 生成されたSQLをレビューする

AIが生成したSQLは、 **常に正しいとは限らない** 。

- 要件と違うSQLが生成されることがある
- パフォーマンスに問題があるSQLが生成されることがある
- セキュリティ上の問題があるSQLが生成されることがある

**レビューするためには、SQLの基礎知識が必要** である。

### 3. データベース設計

データベースの設計（ER図の作成、テーブル構成の決定）は、 **業務の理解がなければできない** 。

- どのようなデータをどのテーブルに保存するか
- テーブル同士の関係をどう定義するか
- データの整合性をどう保つか

これらはAIに丸投げできない。チーム開発では、データベース設計の議論に参加する力が求められる。

## 3. 本教材の学習の進め方

### 構文暗記ではなく「なぜ必要か」から理解する

本教材では、各章の冒頭で **「この技術がないと何が困るのか」** を体験してもらう。
「なぜ必要か」を理解した上で構文を学ぶことで、記憶に残りやすくなる。

### OneCompilerで実際に動かしながら学ぶ

教材に埋め込まれたOneCompilerエディタで、 **SQLを実際に実行しながら** 学習を進める。
「読むだけ」ではなく「動かして確認する」ことで、理解が深まる。

### 全9章の学習マップ

本教材は、以下の3つのフェーズで構成されている。

| フェーズ | 章 | 学習内容 |
|---------|-----|---------|
| **基礎**（Day 5） | 第1〜3章 | DBの必要性、テーブル設計、データ操作（INSERT/UPDATE/DELETE） |
| **SELECT**（Day 10） | 第4〜6章 | データ取得、絞り込み・並び替え、集計 |
| **設計**（Day 15） | 第7〜9章 | 正規化、テーブル結合（JOIN）、ER図 |

:::tip ポイント
SQLの構文を丸暗記する必要はない。 **「こういうことができる」** という概念を理解しておけば、
細かい構文はAIに聞いたりリファレンスを見たりすればよい。
:::
