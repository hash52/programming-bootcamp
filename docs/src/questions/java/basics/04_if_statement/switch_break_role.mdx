explanation: |
  switch文でbreakを省略すると、一致したcaseの処理が実行された後、次のcaseの処理もそのまま実行される。これを **フォールスルー（fall-through）** と呼ぶ。意図しないフォールスルーはバグの原因となるため、各caseの末尾にbreakを書くのが基本である。
