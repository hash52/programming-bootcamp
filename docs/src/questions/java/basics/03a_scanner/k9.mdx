## 問題

nextInt()を使った後にnextLine()を使う場合の注意点として、最も適切なものを選べ。

## 解説

nextInt()は**数値のみを読み取り、改行文字は残す**ため、次にnextLine()を使う場合は注意が必要である。

**なぜこの問題が起きるのか**：

ユーザーが`25`と入力してEnterキーを押すと、以下のようなデータが標準入力に送られる：
```
25\n
```
（`\n`は改行文字）

`nextInt()`は`25`のみを読み取り、`\n`は残る。
次に`nextLine()`を呼ぶと、残った`\n`を読み取ってしまい、空文字列が返される。

<OneCompilerCodeBlock language="java" code={`import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("年齢を入力: ");
        int age = scanner.nextInt();
        scanner.nextLine();  // 改行文字を消費

        System.out.print("名前を入力: ");
        String name = scanner.nextLine();  // 正しく読み取れる

        System.out.println("年齢: " + age);
        System.out.println("名前: " + name);
    }
}`} />

**覚え方**：
- `nextInt()`の後に`nextLine()`を使う場合
  → 間に`scanner.nextLine()`を1回挟む
- `nextInt()`の後に`nextInt()`や`nextDouble()`を使う場合
  → 何も対処は不要（これらのメソッドは空白や改行を自動でスキップする）
