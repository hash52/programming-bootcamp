  **nextLine()とnextInt()の違い**：

  | メソッド | 読み取る内容 | 戻り値の型 |
  |----------|-------------|-----------|
  | `nextLine()` | 1行全体を文字列として読み取る | String |
  | `nextInt()` | 整数のみを読み取る | int |
  | `nextDouble()` | 小数のみを読み取る | double |

  **重要な違い**：
  - `nextLine()`は**改行文字まで**読み取り、改行文字を消費する
  - `nextInt()`は**数値のみ**を読み取り、改行文字は残る

  ```java
  Scanner scanner = new Scanner(System.in);

  // "25"と入力してEnter
  int age = scanner.nextInt();  // 25が読み取られる（改行文字は残る）

  // "田中太郎"と入力してEnter
  String name = scanner.nextLine();  // "田中太郎"が読み取られる
  ```

## 問題

nextLine()とnextInt()の違いとして最も適切なものを選べ。

## 解説

nextLine()とnextInt()は、**読み取るデータの種類が異なる**。

**nextLine()の使い所**：
- 名前や住所など、文字列全体を読み取る
- 数字と文字が混ざった入力（例：「123abc」）
- 空白を含む文字列（例：「田中 太郎」）

**nextInt()の使い所**：
- 年齢や数量など、整数のみを読み取る
- 計算に使う数値
- ユーザーの選択肢（1, 2, 3など）

**よくあるエラー**：
```java
int age = scanner.nextInt();  // "25"と入力
scanner.nextLine();  // 改行文字を消費（これを忘れると次のnextLine()が空文字列を返す）
String name = scanner.nextLine();  // "田中太郎"と入力
```

このように、`nextInt()`の後に`nextLine()`を使う場合は、間に`scanner.nextLine()`を挟んで改行文字を消費する必要がある。
