hint: |
  メソッドのシグネチャに `throws 例外型` を追加すると、例外をキャッチせず呼び出し元に伝搬できる。
  呼び出し元（今回はmain）で `try-catch` する。
  `parseAndDouble` メソッドは `throws NumberFormatException` を宣言し、変換に失敗したらそのまま例外を伝搬させる。
sampleAnswer: |
  public class Main {
      static int parseAndDouble(String s) throws NumberFormatException {
          int value = Integer.parseInt(s);
          return value * 2;
      }

      public static void main(String[] args) {
          String[] inputs = {"10", "abc", "20"};

          for (String input : inputs) {
              try {
                  int result = parseAndDouble(input);
                  System.out.println(input + " × 2 = " + result);
              } catch (NumberFormatException e) {
                  System.out.println(input + " は数値ではありません");
              }
          }
      }
  }
explanation: |
  **例外の伝搬**：`throws` 宣言をしてそのメソッド内でキャッチしなければ、例外は呼び出し元に伝わる。
  これにより「変換する責務」と「エラー処理の責務」を分離できる。

  **`throws` の用途**：メソッドが例外を投げる可能性があることを呼び出し元に知らせる。
  `RuntimeException` のサブクラス（非検査例外）は `throws` を書かなくてもよいが、書く方が明示的でわかりやすい。

  **呼び出しチェーン**：A → B → C の呼び出し順で C で例外が発生した場合、B で `throws` 宣言があれば A まで伝搬できる。

## 問題

文字列を整数に変換して2倍にする `parseAndDouble` メソッドを定義し、例外を呼び出し元（main）に伝搬させるプログラムを実装せよ。

**出力**：
```
10 × 2 = 20
abc は数値ではありません
20 × 2 = 40
```

**要件**：
- `parseAndDouble` メソッドに `throws NumberFormatException` を宣言すること
- メソッド内では `try-catch` せず、例外を呼び出し元に伝搬させること
- `main` で `try-catch` すること
