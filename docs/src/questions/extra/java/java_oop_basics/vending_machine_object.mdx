sampleAnswer: |
  class VendingMachine {
      String itemName;
      int price;
      int stock;
      int sales;

      int buy(int money) {
          if (stock <= 0) {
              System.out.println("在庫がありません");
              return -1;
          }
          if (money < price) {
              System.out.println("お金が足りません");
              return -1;
          }
          stock--;
          sales += price;
          return money - price;
      }

      void status() {
          System.out.println(itemName + " " + price + "円 在庫:" + stock + " 売上:" + sales + "円");
      }
  }

  public class Main {
      public static void main(String[] args) {
          VendingMachine vm = new VendingMachine();
          vm.itemName = "コーラ";
          vm.price = 150;
          vm.stock = 3;
          vm.sales = 0;

          int change = vm.buy(200);
          System.out.println("お釣り: " + change + "円");
          vm.buy(100);
          vm.status();
      }
  }
explanation: |
  **複数条件のチェック**：在庫チェックと金額チェックを順に行い、問題があれば早期リターンする。
  この「ガード節」パターンはネストを浅く保つのに有効である。

  **複数フィールドの更新**：`buy()` で `stock--` と `sales += price` を同時に更新する。
  1つのメソッドが複数のフィールドを変更するのは自然な設計である。

  **int の戻り値**：お釣りをint型で返すことで呼び出し元が値を使える。エラーケースは -1 を返す慣例がある。

## 問題

自動販売機を表す `VendingMachine` クラスを定義し、在庫管理と購入処理を実装せよ。

**出力**：
```
お釣り: 50円
お金が足りません
コーラ 150円 在庫:2 売上:150円
```

**要件**：
- `itemName`・`price`・`stock`・`sales` フィールドを持つこと
- `int buy(int money)` でお釣りを返す（失敗時は -1）こと
- `void status()` で現在の状態を表示すること
