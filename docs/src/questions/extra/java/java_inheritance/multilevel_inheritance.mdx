sampleAnswer: |
  class LivingThing {
      String name;

      LivingThing(String name) {
          this.name = name;
      }

      void breathe() {
          System.out.println(name + "は呼吸している");
      }

      String getInfo() {
          return "生き物: " + name;
      }
  }

  class Animal extends LivingThing {
      String species;

      Animal(String name, String species) {
          super(name);
          this.species = species;
      }

      void move() {
          System.out.println(name + "が移動している");
      }

      @Override
      String getInfo() {
          return super.getInfo() + ", 種: " + species;
      }
  }

  class Dog extends Animal {
      String breed;

      Dog(String name, String breed) {
          super(name, "犬");
          this.breed = breed;
      }

      void bark() {
          System.out.println(name + "がワンワン！");
      }

      @Override
      String getInfo() {
          return super.getInfo() + ", 犬種: " + breed;
      }
  }

  public class Main {
      public static void main(String[] args) {
          Dog dog = new Dog("ポチ", "柴犬");

          System.out.println(dog.getInfo());
          dog.breathe();
          dog.move();
          dog.bark();
      }
  }
explanation: |
  **多段階の継承**：`Dog` は `Animal` を継承し、`Animal` は `LivingThing` を継承している。
  `Dog` は両方のクラスのフィールドとメソッドを全て引き継ぐ。

  **`super.getInfo()` の連鎖**：`Dog.getInfo()` → `super.getInfo()`（Animal）→ `super.getInfo()`（LivingThing）のように連鎖して情報を積み上げる。

  **継承の深さとトレードオフ**：深い継承ツリーはコードの再利用を促すが、複雑さも増す。実際の設計では2〜3段階程度にとどめることが多い。


- 3段階の継承チェーンを実装すること
