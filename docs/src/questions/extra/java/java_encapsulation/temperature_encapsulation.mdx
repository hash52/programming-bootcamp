hint: |
  `temperature` フィールドを `private` にし、setter でバリデーションを行う。
  摂氏の絶対零度（-273.15度）未満は拒否する。
  摂氏↔華氏の変換メソッドも提供する（°F = °C × 9/5 + 32）。
sampleAnswer: |
  class Temperature {
      private double celsius;

      Temperature(double celsius) {
          if (celsius < -273.15) {
              throw new IllegalArgumentException("絶対零度未満の温度は設定できません: " + celsius);
          }
          this.celsius = celsius;
      }

      public double getCelsius() { return celsius; }

      public void setCelsius(double celsius) {
          if (celsius < -273.15) {
              System.out.println("無効な温度です（絶対零度未満）: " + celsius);
              return;
          }
          this.celsius = celsius;
      }

      public double getFahrenheit() {
          return celsius * 9.0 / 5.0 + 32;
      }

      public void display() {
          System.out.printf("%.1f°C / %.1f°F%n", celsius, getFahrenheit());
      }
  }

  public class Main {
      public static void main(String[] args) {
          Temperature t = new Temperature(25.0);
          t.display();

          t.setCelsius(100.0);
          t.display();

          t.setCelsius(-300.0);  // 無効
          t.display();           // 変更されていない

          Temperature boiling = new Temperature(100.0);
          System.out.println("沸点: " + boiling.getCelsius() + "°C");
      }
  }
explanation: |
  **バリデーション付き setter**：摂氏-273.15度（絶対零度）未満の値は物理的に存在しないため、setter で拒否する。

  **変換メソッド**：`getFahrenheit()` は getter の一種だが、計算結果を返す。フィールドとして保持する必要はなく、常に `celsius` から導出できる。

  **コンストラクタの検証**：コンストラクタでも検証を行うことで、オブジェクト生成時から有効な状態を保証する。

## 問題

`Temperature` クラスで摂氏温度をカプセル化し、絶対零度（-273.15°C）未満の値を拒否するプログラムを実装せよ。

**出力**：
```
25.0°C / 77.0°F
100.0°C / 212.0°F
無効な温度です（絶対零度未満）: -300.0
100.0°C / 212.0°F
沸点: 100.0°C
```

**要件**：
- `celsius` を `private` にすること
- setter で-273.15未満の値を拒否すること
- 華氏変換メソッド `getFahrenheit()` を実装すること
