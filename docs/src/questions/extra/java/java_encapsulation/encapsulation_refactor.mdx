hint: |
  公開されているフィールドを `private` に変更し、getter/setter を追加する。
  setter にバリデーションを追加することで、不正な値の代入を防ぐ。

sampleAnswer: |
  class Product {
      private String name;
      private int price;
      private int stock;

      Product(String name, int price, int stock) {
          this.name = name;
          setPrice(price);
          setStock(stock);
      }

      public String getName() { return name; }
      public int getPrice() { return price; }
      public int getStock() { return stock; }

      public void setPrice(int price) {
          if (price < 0) {
              System.out.println("価格は0以上でなければなりません");
              return;
          }
          this.price = price;
      }

      public void setStock(int stock) {
          if (stock < 0) {
              System.out.println("在庫数は0以上でなければなりません");
              return;
          }
          this.stock = stock;
      }

      public boolean sell(int quantity) {
          if (quantity > stock) {
              System.out.println("在庫不足です（在庫: " + stock + "）");
              return false;
          }
          stock -= quantity;
          return true;
      }

      public void display() {
          System.out.println(name + ": " + price + "円 (在庫: " + stock + ")");
      }
  }

  public class Main {
      public static void main(String[] args) {
          Product p = new Product("りんご", 150, 10);
          p.display();

          p.sell(3);
          p.display();

          p.setPrice(-100);  // 無効
          p.sell(20);        // 在庫不足
          p.display();
      }
  }
explanation: |
  **リファクタリングの目的**：フィールドへの直接アクセスを排除し、メソッド経由でのみ状態を変更できるようにする。

  **`sell()` メソッドへの集約**：在庫の減算ロジックを `main` ではなくクラス内に移動することで、ビジネスロジックが一箇所にまとまる。

  **setter のバリデーション**：`setPrice(-100)` が拒否されることで、`product.price = -100;` のような不正操作が防止できる。

```
価格は0以上でなければなりません
在庫不足です（在庫: 7）
りんご: 150円 (在庫: 7)
```

- すべてのフィールドを `private` にすること
- `setPrice()` で負の値を拒否すること
