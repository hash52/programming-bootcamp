hint: |
  setter を定義しなければ、外部からフィールドへの書き込みができなくなる（read-only）。
  コンストラクタで初期化し、getter のみで値を読み出せるようにする。

sampleAnswer: |
  class ImmutableDate {
      private final int year;
      private final int month;
      private final int day;

      ImmutableDate(int year, int month, int day) {
          this.year = year;
          this.month = month;
          this.day = day;
      }

      // getter のみ（setter なし）
      public int getYear() { return year; }
      public int getMonth() { return month; }
      public int getDay() { return day; }

      public String toString() {
          return String.format("%d年%02d月%02d日", year, month, day);
      }

      public boolean isLeapYear() {
          return (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
      }
  }

  public class Main {
      public static void main(String[] args) {
          ImmutableDate d1 = new ImmutableDate(2024, 3, 15);
          ImmutableDate d2 = new ImmutableDate(2025, 1, 1);

          System.out.println(d1.toString());
          System.out.println("年: " + d1.getYear());
          System.out.println("うるう年: " + d1.isLeapYear());

          System.out.println(d2.toString());
          System.out.println("うるう年: " + d2.isLeapYear());
      }
  }
explanation: |
  **getter のみのフィールド（read-only）**：setter を定義しなければ外部から値を変更できない。
  コンストラクタで一度だけ設定する設計。

  **`final` フィールド**：`final` をつけると、コンストラクタでの初期化後は変更不可になる。
  コンパイラがチェックするため、うっかり変更してしまうミスを防げる。

  **不変オブジェクトの利点**：スレッドセーフ、値のコピーが不要、予期しない変更がない。`String`, `Integer` など標準ライブラリも不変クラスが多い。

## 問題

`ImmutableDate` クラスを実装し、日付を read-only なオブジェクトとして表現するプログラムを作れ。

**出力**：
```
2024年03月15日
年: 2024
うるう年: true
2025年01月01日
うるう年: false
```

**要件**：
- フィールドを `private final` にすること
- getter のみ定義し setter は定義しないこと
- `isLeapYear()` でうるう年を判定すること（4で割り切れるかつ100で割り切れない、または400で割り切れる）
