hint: |
  setter でフィールドに値をセットする前に検証（バリデーション）を行う。
  不正な値なら警告メッセージを出して設定しない（または例外をスロー）。
  例：年齢が負なら拒否、スコアが0〜100の範囲外なら拒否。
sampleAnswer: |
  class Student {
      private String name;
      private int score;
      private int age;

      public String getName() { return name; }
      public int getScore() { return score; }
      public int getAge() { return age; }

      public void setName(String name) {
          if (name == null || name.isEmpty()) {
              System.out.println("名前は空にできません");
              return;
          }
          this.name = name;
      }

      public void setScore(int score) {
          if (score < 0 || score > 100) {
              System.out.println("スコアは0〜100の範囲で設定してください: " + score);
              return;
          }
          this.score = score;
      }

      public void setAge(int age) {
          if (age < 0 || age > 150) {
              System.out.println("無効な年齢です: " + age);
              return;
          }
          this.age = age;
      }

      public void display() {
          System.out.println(name + " (年齢:" + age + ", スコア:" + score + ")");
      }
  }

  public class Main {
      public static void main(String[] args) {
          Student s = new Student();
          s.setName("田中");
          s.setAge(20);
          s.setScore(85);
          s.display();

          s.setScore(150);   // 無効
          s.setAge(-5);      // 無効
          s.setName("");     // 無効
          s.display();      // 変更されていないことを確認
      }
  }
explanation: |
  **setter でのバリデーション**：値の設定前に検証することで、オブジェクトを常に有効な状態に保てる。
  フィールドへの直接アクセスでは検証できないが、setter を使えば必ず検証を通る。

  **早期 return パターン**：条件を満たさない場合に `return;` でメソッドを抜けることで、フィールドへの代入を防ぐ。

  **カプセル化の本質**：「データの保護」がカプセル化の重要な目的。setter が「門番」の役割を果たす。

## 問題

`Student` クラスの setter にバリデーションを追加し、不正な値を拒否するプログラムを実装せよ。

**出力**：
```
田中 (年齢:20, スコア:85)
スコアは0〜100の範囲で設定してください: 150
無効な年齢です: -5
名前は空にできません
田中 (年齢:20, スコア:85)
```

**要件**：
- `setScore()` で0〜100の範囲チェックをすること
- `setAge()` で0〜150の範囲チェックをすること
- `setName()` でnullおよび空文字チェックをすること
