hint: |
  フィールドに `private` をつけると、そのクラスの外からアクセスできなくなる。
  `public String name;` → 外部からアクセス可能
  `private String name;` → 外部からアクセス不可（同じクラスのメソッドからはアクセス可能）
sampleAnswer: |
  class SecureBox {
      private String secret;
      private int accessCount = 0;

      void storeSecret(String value) {
          this.secret = value;
          System.out.println("シークレットを保存しました");
      }

      String revealSecret(String password) {
          if ("1234".equals(password)) {
              accessCount++;
              return secret;
          } else {
              return "パスワードが違います";
          }
      }

      int getAccessCount() {
          return accessCount;
      }
  }

  public class Main {
      public static void main(String[] args) {
          SecureBox box = new SecureBox();
          box.storeSecret("TOP SECRET");

          // box.secret = "HACK"; // private なので外からアクセス不可（コンパイルエラー）

          System.out.println(box.revealSecret("wrong"));
          System.out.println(box.revealSecret("1234"));
          System.out.println(box.revealSecret("1234"));
          System.out.println("アクセス回数: " + box.getAccessCount());
      }
  }
explanation: |
  **`private` アクセス修飾子**：フィールドやメソッドをクラスの外部から隠す。
  `private String secret;` とすると `box.secret` はコンパイルエラーになる。

  **カプセル化の目的**：データへの直接アクセスを禁止し、メソッドを通じてのみ操作させることで、不正な変更を防ぐ。

  **内部からはアクセス可能**：`private` でも、そのクラスのメソッド内からは `this.secret` でアクセスできる。
---


## 問題

`private` フィールドを持つ `SecureBox` クラスを実装し、パスワード認証でのみシークレットを取り出せるようにせよ。

**出力**：
```
シークレットを保存しました
パスワードが違います
TOP SECRET
TOP SECRET
アクセス回数: 2
```

**要件**：
- `secret` と `accessCount` フィールドを `private` にすること
- パスワード `"1234"` が正しい場合のみ `secret` を返すこと
