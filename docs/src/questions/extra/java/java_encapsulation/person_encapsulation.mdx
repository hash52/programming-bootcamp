hint: |
  全フィールドを `private` にし、コンストラクタで初期化する。
  全フィールドに getter を定義し、変更が必要なフィールドだけに setter を定義する。
  `name` は初期化後変更不可（getter のみ）、`email` は変更可能（getter + setter）にする。
sampleAnswer: |
  class Person {
      private final String name;  // 変更不可
      private int age;
      private String email;

      Person(String name, int age, String email) {
          this.name = name;
          this.age = age;
          this.email = email;
      }

      // getter（全フィールド）
      public String getName() { return name; }
      public int getAge() { return age; }
      public String getEmail() { return email; }

      // setter（変更可能なフィールドのみ）
      public void setAge(int age) {
          if (age >= 0) this.age = age;
      }

      public void setEmail(String email) {
          if (email != null && email.contains("@")) {
              this.email = email;
          } else {
              System.out.println("無効なメールアドレスです");
          }
      }

      public void display() {
          System.out.println(name + " | " + age + "歳 | " + email);
      }
  }

  public class Main {
      public static void main(String[] args) {
          Person p = new Person("田中太郎", 25, "tanaka@example.com");
          p.display();

          p.setAge(26);
          p.setEmail("tanaka.new@example.com");
          p.display();

          p.setEmail("invalid-email");  // 無効
          p.display();
      }
  }
explanation: |
  **完全カプセル化**：全フィールドを `private` にし、コンストラクタとgetter/setterのみで外部とやり取りする設計。

  **`final` で変更不可**：`name` に `final` をつけることで、コンストラクタ以降の変更を型レベルで防ぐ。

  **setter のバリデーション**：`setEmail` でメールアドレスの簡易チェック（`@` を含むか）を行う。

## 問題

`Person` クラスを完全にカプセル化し、`name` は変更不可、`age`/`email` は検証付き setter で変更可能にするプログラムを作れ。

**出力**：
```
田中太郎 | 25歳 | tanaka@example.com
田中太郎 | 26歳 | tanaka.new@example.com
無効なメールアドレスです
田中太郎 | 26歳 | tanaka.new@example.com
```

**要件**：
- `name` は `private final` にし getter のみ定義すること
- `setEmail()` で `@` を含まないアドレスを拒否すること
