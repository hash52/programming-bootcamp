explanation: |
  複数テーブルのJOINと `WHERE` による絞り込みを組み合わせることで、
  複雑な条件のデータを取得できる。

  **この問題の解法**：
  1. 「アサインされている社員」→ `employees` と `assignments` を INNER JOIN
  2. 「開発部に所属」→ `employees` と `departments` を INNER JOIN して `WHERE dept_name = '開発部'`

  **INNER JOIN の自動フィルタリング**：
  - `INNER JOIN assignments a ON e.emp_id = a.emp_id` は
    `assignments` に存在する社員のみを返す（アサインされていない社員は除外）
  - これにより「アサインされている社員のみ」が自動的に絞り込まれる

  **JOINとWHEREの組み合わせ**：
  JOINで関連テーブルを結合した後、WHEREで条件を絞り込む。
  JOINはテーブルを繋ぐ、WHEREは結果を絞り込む、という役割分担を意識すると良い。
hint: "3テーブルをJOINした後にWHEREで絞り込む順序を考えよう"
