explanation: |
  **ブランチ（branch）** は、コミット履歴を分岐させることで、複数の作業を安全に並行して進める仕組みである。

  **ブランチを使う理由**：
  - `main` ブランチを安定した状態に保ちながら、新機能の開発や修正を行える
  - 複数のメンバーが互いの作業を妨げずに同時に開発できる
  - 開発した機能を `main` にマージする前にレビューできる

  **ブランチのイメージ**：
  ```
  main:    A --- B --- C
                  \
  feature:         D --- E
  ```
  - `feature` ブランチは `B` から分岐し、`D`・`E` という独自の変更を持つ
  - `feature` での変更が `main` に影響しない

  **代表的なブランチ**：
  - `main` / `master`: 本番リリース用の安定ブランチ
  - `feature/xxx`: 新機能開発用ブランチ
  - `fix/xxx`: バグ修正用ブランチ
---

Gitのブランチの説明として最も適切なものを選べ。
