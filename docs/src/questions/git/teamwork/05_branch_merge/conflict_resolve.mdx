explanation: |
  コンフリクト（競合）は、同じ箇所が複数のブランチで別々に変更されたときに発生する。

  **コンフリクトマーカーの見方**：
  ```
  <<<<<<< HEAD（現在のブランチ）
  int result = a + b;
  =======
  int result = a * b;
  >>>>>>> feature/calculator（マージするブランチ）
  ```

  **解決手順**：
  1. コンフリクトが発生したファイルを開く（`git status` で確認）
  2. マーカー（`<<<<<<<`, `=======`, `>>>>>>>`）を見つける
  3. 残したいコードを選択し、マーカーを含む不要な行を削除する
  4. ファイルを保存する
  5. `git add ファイル名` でステージに追加する
  6. `git commit` でマージを完了させる

  **コンフリクトを防ぐ方法**：
  - こまめに `git pull` してリモートの変更を取り込む
  - 同じファイルの同じ箇所を複数人が変更しないよう作業を分担する
---

マージコンフリクトが発生した場合の解決手順として正しいものを選べ。
