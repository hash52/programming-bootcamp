explanation: |
  **Git** は、ファイルの変更履歴を管理するための **分散型バージョン管理システム** である。

  Gitを使うと：
  - ファイルの変更履歴を **時系列で記録** できる
  - いつでも **任意の時点の状態に戻せる**
  - **誰がいつ何を変えたか** が一目でわかる
  - 複数人の変更を **安全に統合** できる

  **よくある間違い**：
  - Cの「ファイルを公開するサービス」はGitHubの説明に近い。Git自体はローカルで動作するツールである
  - GitとGitHubは異なるものである。GitはツールでありGitHubはGitリポジトリのホスティングサービスである
