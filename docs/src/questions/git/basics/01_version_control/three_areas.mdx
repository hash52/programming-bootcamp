explanation: |
  Gitでは、ファイルの状態が以下の **3つのエリア** を順番に移動する。

  1. **作業ディレクトリ（Working Directory）**: 実際にファイルを編集する場所
  2. **ステージ（Staging Area / Index）**: 次のコミットに含めるファイルを一時的に置く場所
  3. **リポジトリ（Repository）**: コミットとして変更履歴が保存される場所

  流れ：
  - ファイルを **編集** する（作業ディレクトリ）
  - `git add` で **ステージに追加** する
  - `git commit` で **リポジトリに記録** する

  **なぜステージが必要なのか**：
  - 変更したファイルの中から、コミットに含めたいものだけを選べる
  - 複数の変更をまとめて1つのコミットにしたり、別々のコミットに分けたりできる
