explanation: |
  Gitの基本用語の意味は以下の通りである。

  - **リポジトリ（repository）**: Gitがファイルの変更履歴を保存・管理する場所。`.git` ディレクトリとして実体がある
  - **コミット（commit）**: 変更内容を履歴として記録するスナップショット。「セーブポイント」のイメージ
  - **ブランチ（branch）**: 開発の流れを分岐させる仕組み。並行して複数の作業を進められる
  - **ステージングエリア（staging area）**: コミットに含めるファイルを選択する中間領域

  **よくある混同**：
  - Bの「インターネット上に公開するWebサービス」はGitHubの説明である。リポジトリ自体はローカルにも存在する
  - Cは `git add` の説明であり、コミットではない

Gitにおける **リポジトリ** の説明として正しいものを選べ。
