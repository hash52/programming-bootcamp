explanation: |
  `git diff` と `git diff --staged` の違いは、比較対象である。

  | コマンド | 比較対象 |
  |----------|----------|
  | `git diff` | 作業ディレクトリ ↔ ステージ（未 `git add` の変更） |
  | `git diff --staged` | ステージ ↔ 最新コミット（`git add` 済みで未コミットの変更） |

  **使い分けの例**：
  1. `git diff` → 「どのファイルをどう変更したか」を `git add` 前に確認する
  2. `git add ファイル名` → ステージに追加
  3. `git diff --staged` → 「次のコミットに何が含まれるか」を確認する
  4. `git commit -m "メッセージ"` → コミット

  こまめに `git diff` を確認することで、意図しない変更のコミットを防げる。
