explanation: |
  MVCで役割を分けることにより、変更の影響範囲を最小限に抑えることができる。

  具体的なメリット：
  - **Viewのデザインを変更してもModelのロジックに影響しない**
  - **ModelのビジネスロジックをViewと独立してテストできる**
  - **同じModelを複数のViewで再利用できる**（例：同じデータをHTML表示とCSVダウンロードで使い回す）

  コードの見通しが良くなり、チーム開発での分業もしやすくなる。
