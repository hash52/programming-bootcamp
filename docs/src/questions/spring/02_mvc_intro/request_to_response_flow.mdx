explanation: |
  SpringMVCの処理フローは以下の順で行われる。

  1. ブラウザからHTTPリクエスト送信
  2. **Controller** がリクエストを受信し、処理を指示
  3. **Model**（Serviceなど）がビジネスロジックを実行
  4. Controller がModelの結果を受け取り、Viewに渡す
  5. **View**（Thymeleaf）がHTMLを生成
  6. ブラウザが受け取り、画面を表示

  ControllerがModelとViewの両方と通信する「橋渡し役」である点が重要である。
