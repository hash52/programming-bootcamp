### 詳細な内容

#### 5. プログラムの大原則（161-375行目）

**原則1: Javaではmainメソッドを起点に実行が始まる**（179-193行目）
- mainメソッドがエントリーポイント
- mainメソッドがないと実行できない

**原則2: プログラムは上から下に実行される**（196-228行目）
- 書かれた順番に上から下に実行
- OneCompilerCodeBlockで実行可能なサンプル

**原則3: `{}`でブロックを作り、処理をまとめる**（232-256行目）
- Hello Worldには2つのブロックがある
- 外側（クラス）と内側（mainメソッド）の入れ子構造
- 今は2つのブロックがあることが分かれば十分

**原則4: セミコロン`;`で命令の終わりを示す**（259-284行目）
- 1つの命令の終わりに必ずセミコロン
- セミコロンがあれば改行しなくても良い
- 1行で書くこともできるが、非常に読みにくい

**原則5: インデント（字下げ）で構造を見やすくする**（287-331行目）
- 3つのパターンで比較：
  1. 改行なし（非常に読みにくい）
  2. 改行あり、インデントなし（どこがどのブロックに属するか分かりにくい）
  3. 改行あり、インデントあり（最も読みやすい）
- スペース4つまたはタブ1つで統一

**補足: どこで改行できるか？**（334-349行目）
- 細かいルールは今は覚えなくて良い
- 「とりあえずやってみて、体験して学ぶ」ことがプログラミング上達の近道
- Eclipseがエラーを教えてくれる

**コードの説明（振り返り）**（352-375行目）
- `public class HelloWorld`の説明
- `public static void main(String[] args)`の説明
- `System.out.println("Hello World")`の説明
